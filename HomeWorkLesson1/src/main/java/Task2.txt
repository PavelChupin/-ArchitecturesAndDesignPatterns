Задание №2

Для домашней системы в идеале я бы выбрал клиент серверный варинт:
Почему:
1. Нет проблемы с администрированием
2. Нет проблем с железом и расположением сервера
3. Не требует высоких нагрузок, работать с ситемой будет 1-2 человека.
4. Не требует интеграций со внешними системами.
5. Не требует особо высокой безопасности, сама система такая из начально 
уже обладает повышенной безопасностью, ввиду не доступности сервеной части из вне, 
только от своих клиентских приложений
6. Доваольно быстры вариант, ввиду отсутствия лишних слоев кодов оберток между кодом клиентского 
и серверного прилоения


Минус на мой взягляд:
1. Не удобно разрабатывать/дорабатывать/исправлять ошибки в виду сильной связанности кода 
и его объема в единицу приложения, доработка по времени сильно затратна
2. Не современый тренд )) 
3. Обязательно установка клиентского приложения на устройство. Толстый клиент.


В идеале если бы стоял вопрос начинаем делать такой проект, то все же остановился на 
микросервисном подходе с использованием REST + JSON (База + Сервер + Web 3-х звенка)
1. Удобство доработки, микросервисы мелкие в отличии от монолитного клиент сервера, 
доработка по времени сильно меньше
2. Отсутствие сильно связанного кода в отличии от клиент-сервера
3. Гибкие возможности расширения
4. Хорошие и удобные перспективы интеграции
5. Доступность сервеной части из браузера, настройка гибкого доступа из в не если это необходимо
6. Не требует для клиентского устройста установки и 
таскания тежеловесного клиента (тонкий клиент)
7. Современный тренд 
8. Много внешних ресурсов где можно развернуть приложение. 

Минусы:
1. Особое внимание безопасности, т.к. имеется доступ из внешних клиентских приложений.



SOA и его модификации с XML не рассматриваю, т.к. для текущей задаче это выглядит излишне тежеловестно
